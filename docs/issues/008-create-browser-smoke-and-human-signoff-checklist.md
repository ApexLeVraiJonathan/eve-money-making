# Create Browser Smoke And Human Signoff Checklist

Type: HITL

## Parent

[`../prd/pre-main-tradecraft-acceptance-testing.md`](../prd/pre-main-tradecraft-acceptance-testing.md)

## What to build

Create a local checklist for browser smoke testing, rough performance smoke timings, and final domain-owner signoff. The checklist should make it clear what can be automated by browser tooling and what requires human business judgment.

## Acceptance criteria

- [x] Checklist covers public Tradecraft overview, history, details, and investment pages.
- [x] Checklist covers user Participation state and Rollover Intent controls.
- [x] Checklist covers admin Cycle planning, opening, settlement, and Settlement Report visibility.
- [x] Checklist covers admin Participation, Payment Matching, validation, refunds, payout state, and recovery inspection.
- [x] Checklist covers admin profit, capital, fees, snapshots, and JingleYield pages.
- [x] Checklist includes rough performance smoke timings for core pages and Cycle Settlement action.
- [x] Checklist distinguishes browser-automated checks from domain-owner human signoff.
- [x] Checklist states the blocker threshold for UI and performance issues.
